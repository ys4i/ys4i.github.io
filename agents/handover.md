# Portfolio Site (`<username>.github.io`) 開発引き継ぎメモ

## 1. リポジトリ概要

- 目的：
  - 技術系就職活動で提出可能な「作品ポートフォリオサイト」として利用する。
  - 自身が制作したハードウェア・ソフトウェアプロダクト、写真、GitHubリポジトリを一覧・整理して提示する。
- ホスティング：
  - GitHub Pages（ユーザーサイト）  
  - URL: `https://<username>.github.io/`
- 技術スタック：
  - 静的サイト
  - GitHub Pages 標準の Jekyll テーマ（`minima`）を利用

## 2. 現状

- リポジトリルートに `_config.yml` を作成済み。
  - `theme: minima`
  - `title`, `description` など基本的な項目を設定済み。
- `index.md` を作成し、トップページとして利用中。
  - 自己紹介の簡単な文章あり。
  - 代表的なプロダクトを数件掲載（Markdownベース）。
- README は GitHub 上での説明用として簡易な記述（必要に応じて整理）。

## 3. 今後の方針（サイト構成）

### 3.1 想定ページ構成

- `/`（トップ）
  - 一言キャッチコピー
  - 自己紹介（3〜4行）
  - 代表作（3〜6件）をカード風に表示
- `/projects`
  - プロダクト一覧ページ
  - 各プロダクトのサムネイル画像、概要文、リポジトリ／記事リンクを掲載
- `/about`
  - 学歴・専攻・興味分野
  - スキルセット（言語／ツール／得意領域）
- `/contact`
  - メールアドレス、その他連絡先

※現時点では `index.md` のみ作成済み。`projects`, `about`, `contact` は未整備またはドラフト段階。

### 3.2 コンテンツの書き方ポリシー（就活向け）

各プロダクトは、以下のフォーマットで記載したい：

- プロダクト名（Project Name）
- 概要（何のためのものか、誰のためのものか）
- 自分の担当（回路設計、ファームウェア、筐体設計など）
- 技術スタック（MCU名、使用言語、EDAツール、プロトコルなど）
- 成果・スケール（製作台数、利用者、イベントでの使用実績など）
- GitHubリポジトリ／発表スライド／記事へのリンク
- 写真1〜数枚（`/images/...` に配置）
